# ysd-strategy — 沪深300ETF 交易策略

每个交易日下午 14:00 自动唤醒，获取 K 线数据和最近新闻，由 AI 自主分析判断并执行买卖操作。每个交易日晚 20:00 进行交易复盘（AAR），从实战中总结经验教训，持续优化交易策略。

## 依赖 Skill

- **ysd-300ETF** — 获取沪深300ETF（510300）最近 30 个交易日的 K 线数据。详见 `skills/ysd-300ETF/SKILL.md`。
- **ysd-account** — 股票模拟交易账户管理（买入、卖出、查询）。详见 `skills/ysd-account/SKILL.md`。
- **ysd-news** — 获取财联社早间新闻精选（最近 3 天）。详见 `skills/ysd-news/SKILL.md`。
- **ysd-review** — 交易复盘（AAR），系统性回顾交易决策，将经验沉淀为记忆。详见 `skills/ysd-review/SKILL.md`。

---

## 触发条件

### 交易执行（14:00）
- **时间：** 每个交易日下午 14:00（Asia/Shanghai）
- **非交易日不执行：** 周末、法定节假日跳过
- **判断方式：** 获取 K 线数据后，检查最新一条数据的日期是否为今天。如果不是今天，说明今天非交易日，不执行任何操作

### 交易复盘（20:00）
- **时间：** 每个交易日晚上 20:00（Asia/Shanghai）
- **仅交易日执行：** 非交易日不复盘（因为没有交易可复盘）
- **内容：** 按 AAR 结构对当天交易决策进行系统性复盘，详见 `skills/ysd-review/SKILL.md`

## AI 自主决策

拿到 K 线数据、最近新闻和当前账户状态后，AI 自行分析，自主决定：

- **买入、卖出、还是空仓观望**
- **买入/卖出多少数量**

AI 可以自由使用任何分析方法——不做任何限制。这是模拟账户，鼓励大胆尝试、天马行空。唯一要求是给出**明确的操作决策和理由**。

## 交易规则（硬性约束）

无论 AI 做出什么决策，以下规则不可违反：

| 规则 | 说明 |
|------|------|
| 禁止日内交易 | 当天买入不可当天卖出。若当天已有买入记录，不可执行卖出 |
| 操作价格 | 买入和卖出价格均使用**当日最新收盘价** |
| 最小交易单位 | ETF 最小交易单位为 100 份，买入数量必须是 100 的整数倍 |
| 资金约束 | 买入金额不可超过账户可用资金 |
| 持仓约束 | 卖出数量不可超过当前持仓数量 |
| 仅交易日操作 | 非交易日不执行任何买卖 |

## 执行流程

每次被唤醒时，按以下步骤执行：

```
1. 获取K线数据
   └─ 运行 ysd-300ETF 脚本

2. 判断是否交易日
   └─ 最新数据日期 ≠ 今天 → 记录日志，结束
   └─ 最新数据日期 = 今天 → 继续

3. 获取最近新闻（与步骤 2 确认交易日后并行执行）
   └─ 运行: python skills/ysd-news/fetch_news.py --start_date {3天前日期} --end_date {今天}
   └─ 解析返回的 JSON 新闻列表

4. 查询账户状态
   └─ 调用 query_account() 和 query_operations()

5. AI 自主分析决策
   └─ 综合所有数据（K 线 + 新闻 + 账户），自主判断操作方向和数量

6. 执行操作（遵守交易规则）
   ├─ 买入 → buy(stock_code="510300", stock_name="华泰柏瑞沪深300ETF", price=收盘价, quantity=数量)
   ├─ 卖出 → sell(stock_code="510300", price=收盘价, quantity=数量)
   └─ 观望 → 不操作

7. 记录结果
   └─ 将分析过程、决策理由和操作结果写入 memory/YYYY-MM-DD.md
```

### 交易复盘流程（20:00 触发）

```
1. 收集当日交易数据
   ├─ 读取 memory/YYYY-MM-DD.md 获取 14:00 策略执行记录
   ├─ 获取当日完整 K 线数据
   └─ 查询当前账户状态

2. AAR 结构化复盘
   ├─ 交易计划：回顾今天的分析判断和操作计划
   ├─ 实际结果：记录市场走势和操作执行情况
   ├─ 做对了什么：提炼有效的分析逻辑和操作亮点
   └─ 哪里能更好：剖析误判和不足，找到根因

3. 量化评估
   ├─ 当日盈亏、累计收益率
   └─ 自评打分（决策质量/执行纪律/风险控制，1-5 分）

4. 提炼交易洞察
   └─ 总结 1-3 条核心经验教训 + 明日关注事项

5. 持久化保存
   ├─ 运行: python skills/ysd-review/save_review.py --date YYYY-MM-DD --data '{JSON}'
   ├─ 将复盘日志追加到 memory/YYYY-MM-DD.md
   └─ 将核心洞察存入 OpenClaw 记忆（让 AI 持续成长）
```

## 日志记录

每次执行后，将以下信息记录到 `memory/YYYY-MM-DD.md`：

```
## 300ETF 策略执行 (HH:MM)

- 最新收盘价：X.XXX
- 分析摘要：（AI 的分析要点，格式自由）
- 决策：买入 / 卖出 / 观望
- 理由：（为什么做出这个决策）
- 操作：买入 XXX 份 @ X.XXX / 卖出 XXX 份 @ X.XXX / 未操作
- 账户：可用资金 XXXX，持仓 XXX 份，总资产 XXXX
```

## 备注

- 本策略为模拟交易，不涉及真实资金，鼓励大胆尝试
- AI 应从历史操作中学习，回顾之前的决策是否正确，持续优化判断
- **交易复盘是持续进化的核心引擎**——通过 AAR 将实战经验固化为记忆，AI 在后续交易决策中主动参考历史复盘，形成"交易→复盘→改进→交易"的正向闭环
- 复盘数据保存为 JSON（`data/reviews/`）便于导出量化分析，同时写入 OpenClaw 记忆驱动 AI 策略进化
