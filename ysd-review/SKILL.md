# ysd-review — 交易复盘（AAR）

专业的沪深300ETF交易复盘 Skill。采用 AAR（After Action Review）结构化方法，对每个交易日的决策过程、市场判断和执行结果进行系统性回顾，持续优化交易策略。

## 触发条件

- **时间：** 每个交易日晚上 20:00（Asia/Shanghai）
- **仅交易日执行：** 非交易日（周末、节假日）不复盘
- **判断方式：** 与交易策略共享交易日判断逻辑——当天 14:00 有策略执行记录，则 20:00 触发复盘

---

## 复盘结构（AAR 四大问题 × 交易场景）

| 序号 | AAR 问题 | 交易复盘落地 |
|------|----------|-------------|
| 1 | **交易计划** — 今天打算怎么操作？ | 回顾 14:00 策略执行时的分析判断和交易计划（买入/卖出/观望的预期及理由） |
| 2 | **实际结果** — 市场实际怎么走的？ | 对比盘中走势与收盘数据，记录实际执行的操作和当日盈亏情况 |
| 3 | **做对了什么** — 哪些判断和操作值得保持？ | 识别正确的分析逻辑、有效的技术信号、良好的仓位管理等 |
| 4 | **哪里能更好** — 哪些判断失误或执行不当？ | 分析误判原因、遗漏的信号、仓位不合理等，提出改进方案 |

### 重点复盘维度

每次复盘应覆盖以下维度：

- **决策质量：** 买入/卖出/观望的决策逻辑是否合理？信息是否充分？
- **市场判断：** 对趋势、支撑/阻力、成交量等的判断是否准确？
- **风险控制：** 仓位管理是否得当？是否遵守了交易纪律？
- **策略有效性：** 当前使用的分析方法是否有效？参数是否需要调整？
- **情绪管理：** 决策是否受到恐惧/贪婪等情绪干扰？（AI 视角：分析是否存在过度拟合或偏见）

## 执行流程

```
1. 收集当日交易数据
   ├─ 读取 memory/YYYY-MM-DD.md 获取 14:00 策略执行记录
   ├─ 获取当日完整 K 线数据（开盘、最高、最低、收盘、成交量）
   └─ 查询当前账户状态（持仓、资金、总资产）

2. AAR 结构化复盘
   ├─ 交易计划：回顾今天的分析判断和操作计划
   ├─ 实际结果：记录市场实际走势和操作执行情况
   ├─ 做对了什么：提炼有效的分析逻辑和操作亮点
   └─ 哪里能更好：剖析误判和不足，找到根因

3. 量化评估
   ├─ 当日盈亏（持仓市值变化）
   ├─ 决策评分（自评 1-5 分：决策质量、执行纪律、风险控制）
   └─ 累计收益率跟踪

4. 提炼交易洞察
   ├─ 总结 1-3 条核心经验教训
   └─ 制定具体的改进行动（明天交易时应该注意什么）

5. 持久化保存
   ├─ 运行: python skills/ysd-review/save_review.py --date YYYY-MM-DD --data '{JSON}'
   │   → 保存到 data/reviews/YYYY-MM-DD.json（方便导出分析）
   ├─ 将复盘日志追加到 memory/YYYY-MM-DD.md
   └─ 将核心洞察存入 OpenClaw 记忆（让 AI 持续成长）
```

## 数据保存

### JSON 导出（方便导出和回溯分析）

运行 Python 脚本将复盘结果保存到 `data/reviews/` 目录：

```bash
python skills/ysd-review/save_review.py --date 2026-03-24 --data '{
  "market_data": {
    "open": 3.876,
    "high": 3.901,
    "low": 3.852,
    "close": 3.889,
    "volume": 285600000,
    "change_pct": -0.82
  },
  "trade_plan": "基于均线多头排列和放量信号，计划在14:00买入500份",
  "actual_result": "执行买入500份@3.889，收盘价较开盘小幅上涨",
  "operation": {
    "action": "buy",
    "quantity": 500,
    "price": 3.889
  },
  "account": {
    "cash": 8055.50,
    "position": 500,
    "total_asset": 10000.00
  },
  "good": ["正确识别了均线多头排列信号", "买入时机把握较好，盘中未追高"],
  "improve": ["对成交量判断偏乐观，实际放量幅度不如预期", "仓位偏重，首次建仓可以更保守"],
  "lessons": ["首次建仓控制在总资产30%以内更稳妥"],
  "next_actions": ["明天重点观察成交量是否持续放大", "如果缩量回调到3.86附近可考虑加仓"],
  "scores": {
    "decision_quality": 4,
    "execution_discipline": 5,
    "risk_control": 3
  }
}'
```

也支持从 stdin 读取 JSON 数据：

```bash
echo '{"trade_plan": "...", "actual_result": "..."}' | python skills/ysd-review/save_review.py --date 2026-03-24
```

### 文件存储路径

```
data/
└── reviews/
    ├── 2026-03-24.json
    ├── 2026-03-25.json
    └── ...
```

### JSON 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| date | string | 交易日期 YYYY-MM-DD |
| time | string | 复盘时间 HH:MM:SS |
| market_data | object | 当日行情（开/高/低/收/量/涨跌幅） |
| trade_plan | string | 交易计划（14:00 时的分析和操作计划） |
| actual_result | string | 实际结果（市场走势和执行情况） |
| operation | object | 实际操作（action/quantity/price），未操作时 action 为 "hold" |
| account | object | 账户状态（cash/position/total_asset） |
| good | string[] | 做对的地方 |
| improve | string[] | 可改进的地方 |
| lessons | string[] | 核心经验教训 |
| next_actions | string[] | 明日关注和改进行动 |
| scores | object | 自评分数（decision_quality/execution_discipline/risk_control，1-5 分） |

### OpenClaw 记忆存储

复盘后的核心洞察必须写入 OpenClaw 记忆，确保 AI 在后续交易中持续进化。写入内容包括：

- **交易经验（lessons）：** 从实战中提炼的交易认知，长期有效
- **策略洞察：** 对分析方法有效性的反思，哪些信号可靠、哪些容易误判
- **改进行动（next_actions）：** 近期需要关注和优化的具体事项

记忆写入格式：

```
## 交易复盘 YYYY-MM-DD

### 交易经验
- [经验1]
- [经验2]

### 策略洞察
- [洞察]

### 明日关注
- [关注点]
```

每次复盘后，将以下内容追加到 `memory/YYYY-MM-DD.md`：

```

### 交易计划
（14:00 策略执行时的分析和操作计划）

### 实际结果
（市场实际走势 + 操作执行情况 + 当日盈亏）

### 做对了什么
- （正确的判断和操作）

### 哪里能更好
- （误判和不足）

### 自评打分
- 决策质量：X/5
- 执行纪律：X/5
- 风险控制：X/5

### 核心洞察
- （最重要的 1-3 条交易经验）

```

## 备注

- 复盘是交易能力持续进化的核心引擎，每个交易日必须执行
- 复盘重点是客观分析决策质量，而非简单看盈亏结果——做对了亏钱也是好决策，做错了赚钱也要反思
- AI 应在每次 14:00 交易决策前，主动回顾最近的复盘记忆，避免重复犯错
- 定期（每周/每月）回顾 `data/reviews/` 中的 JSON 数据，发现系统性规律
